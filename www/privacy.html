Google OAuth & API access

mail-assistant uses Google OAuth 2.0 to access Google Tasks on your behalf. Only the following scope is requested:


ScopePurpose
https://www.googleapis.com/auth/tasks Read, create and update tasks in Google Tasks

No access to Gmail, Google Drive, Google Calendar or any other Google service is requested.

What data is accessed

Google Tasks

  • Read: Open tasks with title and due date — used to detect whether a task has already been created or completed.
  • Write: Creating new tasks (title, priority, due date) and marking tasks as completed.

IMAP emails

  • Email content (subject, sender, body) is fetched from your configured IMAP server and sent to the selected LLM for classification.
  • When using Ollama (local model), email content never leaves your server — all processing happens in your own infrastructure.
  • When using Anthropic or Mistral APIs, email content is sent to that provider’s API. It is the user’s responsibility to ensure compliance with applicable regulations and data processing agreements.

Data stays on your server (Ollama)

When LLM_PROVIDER=ollama is configured:

  • All emails are processed locally on your own server
  • No email data is sent to Anthropic, Mistral or any third party
  • The only outbound connection is to the Google Tasks API

Data stored locally

DataLocationContents
data/seen.db Local SQLite Message-ID, subject, sender, processing status — not email body
data/google_token.pickle Local file Google OAuth token — refreshed automatically
credentials/google_credentials.json Local file Google OAuth client credentials

No data is sent to the application developer. There is no telemetry, no analytics and no cloud backup.

Revoking access

Google access can be revoked at any time via Google Account → Third-party apps with account access.

To delete the local token: rm data/google_token.pickle
